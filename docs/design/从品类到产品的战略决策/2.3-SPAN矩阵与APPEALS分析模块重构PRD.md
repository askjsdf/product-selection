## 一、产品概述

### 1.1 模块定位

**SPAN矩阵 & $APPEALS分析**是AI选品助手中**最核心的决策模块**，位于Step 2（市场与类目研究）的第三个子模块。其作用是：

1. **SPAN矩阵评估**：基于"市场吸引力（Y轴）"和"竞争地位/可进入性（X轴）"对细分市场进行定量评估，输出投资决策建议（GO/NO-GO）
2. **$APPEALS竞品对标**：通过8维度对标分析，找出产品差异化机会点，输出产品规格清单（PRD Summary）
3. **AI决策引擎**：综合SPAN、财务、技术、团队、风险5个维度，给出智能投资建议和关键成功因素（CSF）

### 1.2 用户价值

- **快速决策**：3-5分钟内完成从"品类机会"到"产品规格"的全流程决策
- **数据驱动**：基于30+维度的量化评估，避免主观判断失误
- **执行导向**：直接输出可执行的产品规格、财务预测、风险条件

### 1.3 核心问题诊断

当前UI存在的问题：
1. **信息密度过高**：SPAN + AI决策 + $APPEALS + 差距矩阵 + PRD全部堆叠在一屏，用户找不到重点
2. **交互流程割裂**：SPAN选择→AI决策→$APPEALS对标是三个独立的卡片，缺乏流程引导
3. **视觉层级混乱**：标题、副标题、正文字号不统一，重点不突出
4. **空间浪费严重**：大量的padding和margin导致有效信息区域不足60%

## 二、功能需求定义

### 2.1 SPAN矩阵投资评估工作台

#### 2.1.1 核心功能

**输入**：
- 细分市场列表（来自"宠物用品"→细分品类树的选择）
- 每个细分市场的Y轴评估数据（5维度）
- 每个细分市场的X轴评估数据（4维度）

**处理**：
- 计算Y轴市场吸引力得分（加权平均）
- 计算X轴竞争地位/可进入性得分（加权平均）
- 根据(X, Y)坐标落入四象限（明星/金牛/问题/瘦狗）
- 匹配对应象限的战略指南

**输出**：
- SPAN散点图可视化（ECharts）
- 选中市场的详细评估报告
- 四象限战略建议

#### 2.1.3 交互流程

1. **进入页面**：默认显示所有细分市场的散点图
2. **选择市场**：点击左侧市场卡片 → 右侧散点图高亮对应点 → 下方展开详细评估

4. **查看建议**：根据象限自动匹配战略指南
5. **触发决策**：选中市场的"GO"建议 → 自动触发AI决策报告生成

### 2.2 AI投资评审委员会（Virtual IRB）决策报告

#### 2.2.1 核心功能

**输入**：
- SPAN选中的细分市场评估数据
- 财务可行性数据（预估毛利率、ROI）
- 技术可行性数据（供应链成熟度）
- 团队匹配度数据（资金、运营能力）
- 风险评估数据（白牌风险、市场风险）

**处理**：
- 5维度加权计算综合得分（0-100）
- 基于得分输出决策：GO（≥80）/ SELECTIVE（60-79）/ NO-GO（<60）
- 提取关键成功因素（CSF），按P0/P1/P2优先级排序
- 生成12个月财务预测（分3个阶段）
- 列出附加条件和风险提示

**输出**：
- 综合评分 + GO/NO-GO决策
- 5维度雷达图（SPAN/财务/技术/团队/风险）
- CSF清单（P0/P1/P2优先级）
- 财务预测时间线（3阶段）
- 附加条件和风险提示

3. **查看CSF**：P0/P1/P2按优先级排序，P0用红色标识
4. **查看预测**：财务时间线3阶段卡片，hover显示详细数据
5. **确认决策**：用户点击"接受建议"→进入$APPEALS对标环节

### 2.3 $APPEALS竞品对标工作台

#### 2.3.1 核心功能

**输入**：
- 3个竞品的基础信息（ASIN、价格、评分、销量）
- 8维度评分矩阵（我们 vs 3个竞品 vs 理想产品）
- 每个维度的详细分析（差距原因、改进策略、成本影响）

**处理**：
- 计算8维度加权综合得分
- 识别优势维度（我们 > 竞品平均）
- 识别差距维度（我们 < 竞品或理想）
- 生成差距战略矩阵（填平差距/建立优势/保持持平）
- 输出产品规格清单（PRD Summary）

**输出**：
- 8维度对标评分表
- 价值曲线雷达图（4条线：我们+3竞品）
- 差距战略矩阵（2x2网格）
- 产品规格清单（材料/功能/尺寸/颜色/包装/质保）

1. **选择竞品**：默认选中3个主要竞品（最多3个），用户可取消勾选

6. **导出PRD**：点击"查看完整PRD文档"→跳转到PRD详情页或下载PDF

### 3.2 关键设计原则

1. **渐进式展开**：不是一次性展示所有内容，而是按流程逐步展开
   - 默认只展开"① SPAN矩阵评估"
   - 用户点击"继续下一步"→ 展开"② AI决策报告"，同时折叠①
   - 再次点击"继续下一步"→ 展开"③ $APPEALS对标"，同时折叠②
   - 用户可点击阶段指示器回到任意阶段

2. **左右分栏清晰**：
   - SPAN：左30%（市场列表+详细评估），右70%（散点图）
   - AI决策：全宽单列布局（因为内容需要横向展开）
   - $APPEALS：上全宽（评分表），下左右分栏（雷达图+差距矩阵）

3. **视觉层级明确**：
   - 一级标题（阶段名称）：`font-size-lg`，粗体，带图标
   - 二级标题（卡片标题）：`font-size-md`，粗体
   - 三级标题（维度名称）：`font-size-sm`，粗体
   - 正文内容：`font-size-xs`
   - 辅助说明：`font-size-xs`，`text-tertiary`

4. **颜色编码语义化**：
   - 绿色：优势、建议执行、正向指标
   - 红色：劣势、风险、必须填平
   - 黄色/橙色：中等、警告、选择性执行
   - 灰色：持平、不重要、已完成

5. **数据可视化优先**：
   - SPAN：散点图 > 文字描述
   - AI决策：卡片+进度条 > 纯文字
   - $APPEALS：雷达图+热力表格 > 纯文字

### 4.2 当前HTML结构问题

**现状**：
- SPAN和$APPEALS并排展示在一个页面（第441-700行）
- 使用`.span-section`和`.appeals-section`两个大容器
- 内部又嵌套了多层`.span-content-grid`、`.appeals-content-grid`
- 没有阶段指示器，用户不知道当前在哪一步

**问题**：
1. 信息过载：用户一进来就看到SPAN+AI决策+$APPEALS全部内容
2. 缺乏流程引导：没有"Step 1→Step 2→Step 3"的明确流程
3. 布局固化：grid布局写死宽度（480px、900px），不适配不同屏幕

**需要重构的部分**：
- [ ] 将SPAN、AI决策、$APPEALS拆分为3个独立的section，默认只展开第一个
- [ ] 添加阶段指示器组件（进度条 + 可点击的步骤导航）
- [ ] 使用百分比宽度替代固定像素宽度

### 4.3 当前CSS结构问题

**现状**：
- CSS文件6081行，没有模块化拆分
- 大量冗余样式（如`.panel-title`在多处重复定义）
- padding/margin不统一（有的用`spacing-lg`，有的用`spacing-3xl`）
- 字号不统一（有的`font-size-md`，有的`font-size-sm`，有的`font-size-xs`）

**问题**：
1. 难以维护：修改一个样式可能影响多处
2. 体积臃肿：6000+行CSS加载慢
3. 命名混乱：有`.section-title`也有`.panel-title`，语义重叠

**需要重构的部分**：
- [ ] 提取通用样式到全局变量（如`.stage-header`、`.data-card`）
- [ ] 统一间距系统：只使用`spacing-xs`、`spacing-sm`、`spacing-md`三个级别
- [ ] 统一字号系统：标题用`font-size-md`，正文用`font-size-xs`
- [ ] 考虑CSS模块化（拆分为`span.css`、`appeals.css`、`irb.css`）

### 4.4 当前JS逻辑问题

**现状**：
- `renderSPAN()`、`renderAPPEALS()`、`renderAIDecision()`三个函数独立渲染
- 没有状态管理，用户选择的市场不会联动到AI决策
- 事件监听混乱，有的用`addEventListener`，有的用内联`onclick`

**问题**：
1. 缺乏联动：用户在SPAN选中"智力训练型"→AI决策和$APPEALS应该自动更新为该市场的数据，但当前没有联动
2. 状态丢失：用户刷新页面后，之前的选择全部丢失
3. 代码重复：多处出现相同的DOM操作代码

**需要重构的部分**：
- [ ] 引入状态管理：使用`this.state.currentStage`、`this.state.selectedSegment`
- [ ] 实现数据联动：SPAN选择→触发AI决策渲染→触发$APPEALS渲染
- [ ] 提取公共函数：如`renderCard()`、`renderProgressBar()`

### 4.5 当前数据结构优势

**优点**：
- 数据层与视图层分离良好（`step2-data.js`独立）
- 数据结构完整且合理（SPAN、AI决策、$APPEALS的字段设计科学）

```javascript
switchStage(stageNumber) {
  // 更新state
  this.state.currentStage = stageNumber;

  // 隐藏所有stage-content
  document.querySelectorAll('.stage-content').forEach(content => {
    content.style.display = 'none';
  });

  // 显示目标stage
  const targetStage = document.getElementById(`stage-${stageNumber}-content`);
  if (targetStage) {
    targetStage.style.display = 'block';
  }

  // 更新指示器状态
  document.querySelectorAll('.stage-item').forEach((item, index) => {
    item.classList.remove('active', 'current');
    if (index + 1 < stageNumber) {
      item.classList.add('active'); // 已完成
    } else if (index + 1 === stageNumber) {
      item.classList.add('current'); // 进行中
    }
  });

  // 根据阶段渲染内容
  if (stageNumber === 1) {
    // 已在renderSPAN中完成
  } else if (stageNumber === 2) {
    this.renderAIDecision(spanData.decision);
  } else if (stageNumber === 3) {
    this.renderAPPEALS(appealsData);
  }
}
```
