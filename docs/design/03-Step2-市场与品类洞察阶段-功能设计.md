## 一、阶段定位与目标

### 1.1 业务目标

**核心问题**：在确定品类后，深入分析前，我需要了解这个品类的市场结构是什么样的？

**卖家的真实思考路径**：
```
Step 1已经判断"宠物用品×US×Amazon"值得进入(行业天花板高)
但现在需要回答:
→ 这个品类下有哪些细分产品类型？各占多少销售额？
→ 主流价格带在哪里？我应该做高端还是中端？
→ 主流款式是什么？单碗还是双碗？带摄像头还是不带？
→ 竞品分布如何？Top 10都在做什么？

这些问题的答案,直接决定我在Step 3选择什么产品!
```

**解决方案**：
- 基于Step 1选定的品类，进行品类级市场结构分析
- 提供**产品分类体系**、**价格带分布**、**主流款式洞察**
- 为Step 3的产品选择提供明确的决策依据

**与Step 1行业天花板的区别**：

| 维度 | Step 1 行业天花板 (宏观) | Step 2 市场与品类洞察 (中观) |
|-----|----------------------|------------------------|
| **分析对象** | 整个行业(如美国宠物市场) | 具体品类(如自动喂食器) |
| **核心问题** | 该不该进入这个行业? | 该品类内部结构如何? |
| **数据粒度** | 行业总规模、5年CAGR | 细分类目月销售额、产品分类 |
| **输出** | 进入/不进入的判断 | 产品选择的方向指引 |

### 1.2 输入与输出

| 项 | 内容 |
|-----|------|
| **输入** | Step 1的机会候选池(如"宠物用品×US×Amazon") |
| **处理** | 品类细分 → 产品分类 → 价格带分析 → 主流款式识别 → 机会点标注 |
| **输出** | **品类洞察报告**(为Step 3产品选择提供决策地图) |

### 1.3 关键成功指标(POC演示)

- ✅ 用户能清晰看到品类内部的产品分类结构
- ✅ 用户能理解主流价格带和主流款式
- ✅ 用户能识别"增长机会点"和"红海区域"
- ✅ 为Step 3产品选择提供明确方向

## 2.2 品类细分树

### 2.2.1 功能定位

**作用**：将大品类(如"宠物用品")逐级细分,帮助用户定位到更具体的细分品类

**为什么需要细分?**
- "宠物用品"太宽泛,包含喂食器、玩具、窝垫、清洁用品等数十个子品类
- 不同子品类的市场特征、竞争格局、利润空间差异巨大
- 细分后才能精准进行产品分类分析

## 2.3 产品分类分析 (核心模块) 🆕

### 2.3.1 功能定位与底层逻辑

**为什么这是Step 2的核心**：

在选择具体产品前(Step 3),卖家必须先理解品类内部的产品分类结构,否则就像"盲人摸象"。

**卖家的真实思考路径**：
```
我知道自动喂食器值得做,但:

第一层思考：有哪几种类型？
  → 重力式 vs 电子式 vs 智能式？各占多少销售额？
  → 哪种类型是主流？哪种在快速增长？

第二层思考：价格带分布如何？
  → $10-20 vs $30-60 vs $75-150？
  → 哪个价格带销量最好？竞争如何？

第三层思考：主流款式是什么？
  → 单碗 vs 双碗？带摄像头 vs 不带？
  → 新品机会在哪个细分？

理解这些,我才知道该做"智能款双碗喂食器,定价$45"
而不是盲目做"便宜的重力式喂食器,定价$15"(红海价格战)
```

**三个维度的产品分类**：

```
维度1: 按工作原理/技术类型分类
  → 回答"有哪几种技术路线"
  → 示例: 重力式/电子定时式/智能APP控制式

维度2: 按价格带分类
  → 回答"主流价格区间在哪里"
  → 示例: $10-20低端 / $30-60中端 / $75+高端

维度3: 按功能特征分类
  → 回答"消费者关注哪些功能差异"
  → 示例: 单碗/双碗, 带摄像头/不带, 容量大小
```

## 六、关键设计原则

### 6.1 "先结构,后产品"原则

- ❌ 错误: 直接看350个产品列表,逐个点开看
- ✅ 正确: 先理解品类结构,再有方向地选产品

### 6.2 "数据可视化"原则

- 所有分类数据必须用**柱状图/占比图**展示
- 让用户"一眼看出"主流在哪里、机会在哪里

### 6.3 "策略导向"原则

- 不只是展示数据,而是给出**明确的策略建议**
- 每个分类都要有"推荐/不推荐/谨慎推荐"的结论
