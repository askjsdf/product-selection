# 架构重大修订说明 V2.0

> **修订日期**: 2025-11-18
> **修订原因**: 基于高阶审视，优化业务逻辑连贯性和流程合理性
> **影响范围**: 整体Step架构从5步调整为6步

---

## 一、核心架构变化

### 1.1 Step架构调整

**修订前 (V1.0 - 5步法)**:
```
Step 1: 机会发现 (品类级)
Step 2: 产品选择 (产品级)
Step 3: 利润分析 (深度验证)
Step 4: 市场调研 (竞品分析)
Step 5: 策略决策 (执行规划)
```

**修订后 (V2.0 - 6步法)** 🆕:
```
Step 1: 机会发现 (组合级)
Step 2: 市场与品类洞察 (品类结构分析) ← 🆕 新增
Step 3: 产品选择 (产品级)
Step 4: 利润分析 (深度验证 + 供应链验证)
Step 5: 竞品深度拆解 (差异化策略)
Step 6: 策略决策 (执行规划)
```

### 1.2 修订理由

**问题诊断**:
在原V1.0架构中，Step 2"产品选择"包含了"产品特征分类"分析（如：重力式20% vs 电子式60% vs 智能式18%）。

但存在**逻辑悖论**：
- 用户还没分析品类结构，怎么知道"电子式占60%销售额"？
- 用户还没了解价格带分布，怎么知道"$30-60是主流"？
- 这些信息应该在**选择具体产品之前**就提供给用户！

**真实卖家的思考路径**:
```
1. 选定品类 (Step 1: 宠物用品×US×Amazon)
   ↓
2. 了解品类结构 (Step 2 🆕: 有哪些产品类型？主流价格带？)
   - 重力式20% / 电子式60% / 智能式18%
   - $20-45是主流价格带(65%销售额)
   - 智能式增长最快(+180% YoY)
   ↓
3. 基于洞察选择产品 (Step 3: 我选"智能式+$45-75"方向的产品)
   ↓
4. 利润测算
   ↓
5. 竞品拆解
   ↓
6. 最终决策
```

**解决方案**:
在Step 1和Step 2之间插入**"市场与品类洞察"**阶段，专门分析品类内部结构，为产品选择提供方向指引。

## 三、MCDA应用的三次分层 🆕

### 问题诊断
原架构中MCDA只在Step 1(组合级)和Step 5(最终方案级)应用，Step 2产品选择缺少量化评分。

### 解决方案: MCDA的三次应用

**MCDA v1.0 (Step 1 - 组合级)**:
```
对象: 480个"品类×市场×平台"组合
5维度: MA / CF / PP / RF / RC (完整)
数据源: 9张聚合数据表
输出: 综合得分0-100, Top 20推荐
```

**MCDA v2.0 (Step 3 - 产品级)** 🆕:
```
对象: 细分品类下的具体产品(如50个智能喂食器)
简化3维度:
  - 市场表现 40% (BSR排名 + 月销量估算)
  - 竞争度 30% (评论数 + 平均评分)
  - 利润潜力 30% (价格带 + 预估毛利率)
数据源: 产品级数据(ASIN/SKU)
输出: 产品推荐度评分0-100, 智能排序
```

**MCDA v3.0 (Step 6 - 最终方案级)**:
```
对象: 最终候选方案(产品+策略组合)
完整5维度 + 用户自定义权重
数据源: Step 1-5的全部分析结果
输出: 最终决策排序
```

**为什么需要三次MCDA?**
```
v1.0 (组合级): 从480个组合中筛选 → 快速定位机会
v2.0 (产品级): 从50个产品中排序 → 避免盲目浏览
v3.0 (方案级): 从3个方案中决策 → 系统化最终决策
```

## 六、关键设计原则强化

### 6.1 "先结构,后产品"原则
- Step 2提供品类结构洞察
- Step 3基于结构选择产品
- 避免盲目浏览产品列表

### 6.2 "层级递进"原则
```
Step 1 (宏观): 行业天花板 → 该不该进入这个行业?
Step 2 (中观): 品类结构 → 品类内部有哪些细分?
Step 3 (产品): 具体产品 → 做哪个具体产品?
Step 5 (微观): 竞品拆解 → 竞品怎么做的?
```

### 6.3 "MCDA三次应用"原则
- 组合级 → 产品级 → 方案级
- 避免"只有开头和结尾的评分,中间靠感觉"

### 6.4 "数据驱动"原则
- 每个分析都有明确的数据支撑
- 避免"拍脑袋"决策
