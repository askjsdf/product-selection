## 三维度组合示例

### 有效组合示例
```
✅ {宠物用品 × 美国 × TikTok Shop}
   - 平台在该国家已开通
   - 品类在该平台允许销售
   - 市场数据充足

✅ {智能家居 × 阿联酋 × Noon}
   - 中东高端市场
   - Noon覆盖UAE
   - 智能家居在中东需求旺盛

✅ {服饰配饰 × 泰国 × Shopee}
   - Shopee在泰国是主流平台
   - 服饰是Shopee强势品类
```

### 无效组合示例（会在矩阵中显示为空白）
```
❌ {宠物用品 × 沙特 × Shopee}
   - Shopee未在中东开展业务

❌ {母婴用品 × 德国 × Noon}
   - Noon仅在中东运营

❌ {乐器配件 × 印尼 × TikTok Shop}
   - 某些平台可能限制特定品类销售
```

## 矩阵规模估算

### 品类视图
- 选择1个品类
- 国家维度：25个
- 平台维度：15个
- 最大矩阵：25 × 15 = 375个格子
- 实际有效格子：约150-200个（考虑平台区域覆盖限制）

### 国家视图
- 选择1个国家
- 品类维度：20个
- 平台维度：15个（但会根据该国家筛选）
- 最大矩阵：20 × 15 = 300个格子
- 实际有效格子：约60-120个（取决于该国家有哪些平台）

### 平台视图
- 选择1个平台
- 国家维度：25个（但会根据平台覆盖筛选）
- 品类维度：20个
- 最大矩阵：25 × 20 = 500个格子
- 实际有效格子：约100-200个（取决于平台区域覆盖）

## 数据维护策略

### 平台覆盖矩阵
维护一个"平台×国家"可用性矩阵：

```json
{
  "amazon": ["US", "CA", "MX", "UK", "DE", "FR", "IT", "ES", "NL", "JP", "AU"],
  "tiktok": ["US", "UK", "SG", "MY", "TH", "VN", "PH", "ID"],
  "shopee": ["SG", "MY", "TH", "VN", "PH", "ID", "TW"],
  "noon": ["AE", "SA", "EG"],
  "lazada": ["SG", "MY", "TH", "VN", "PH", "ID"],
  ...
}
```

### 品类限制矩阵
维护一个"平台×品类"限制列表：

```json
{
  "restricted_combinations": [
    {"platform": "tiktok", "category": "health_wellness", "reason": "保健品受限"},
    {"platform": "amazon", "category": "musical_instruments", "country": "DE", "reason": "德国乐器需特殊认证"},
    ...
  ]
}
```

### 数据更新频率
- **平台覆盖数据**：每季度更新（新平台开站）
- **品类限制数据**：每月更新（政策变化）
- **市场数据**：每周/每日更新（热力值计算依赖）

## UI展示建议

### 1. 矩阵热力图尺寸控制

**问题**：25×15可能太大，一屏显示不下

**方案A - 分页展示**：
```
显示前10个国家，[查看更多]展开
或者提供滚动条
```

**方案B - 智能收起**：
```
默认只显示"有数据"的格子
隐藏全空白的行/列
提供"展开全部"按钮
```

**方案C - 分组折叠**：
```
国家按区域分组，可折叠
- 北美 (3个) [展开]
- 欧洲 (7个) [展开]
- 东南亚 (6个) [展开]
```

**推荐：方案B + 方案C结合**

### 2. 空白格子处理

```
有数据的格子：显示热力颜色 + 评分
平台未覆盖：灰色 + "-"
品类受限：灰色 + "🚫"
即将开通：淡蓝色 + "Coming Soon"
```

### 3. 排序功能

**国家/平台维度支持排序**：
- 按字母顺序
- 按平均评分（该行/列的平均值）
- 按最高评分
- 按区域分组

## 总结

这套三维度数据体系：

1. **完整性**：覆盖主流跨境电商场景
2. **可扩展**：易于新增品类/国家/平台
3. **可维护**：分层结构清晰
4. **可计算**：支持MCDA算法对每个组合评分
